# 财务业务知识库

本文档整合财务模块核心业务知识，供AI理解和处理财务相关问题。

---

## 一、核心概念定义

### 1.1 采购与销售类型

| 类型 | 定义 | 货物所有权 | 收入确认方式 |
|------|------|-----------|-------------|
| **寄售** | 不买断上游货物，待下游使用结算时再向上游买断 | 归上游 | 通常净额法 |
| **经销** | 买断上游货物，取得销售与处置权利 | 归我方 | 通常总额法 |

### 1.2 收入确认方式

| 方式 | 定义 | 适用场景 |
|------|------|----------|
| **总额法** | 对货品形成实际控制，按下游全额付款记录收入 | 经销、仓库存储、即入即出 |
| **净额法** | 对货品未形成实际控制，按上下游差价记录收入 | 寄售院端配送、GPO医院指定供应商 |

### 1.3 配送方式

| 方式 | 定义 | 是否经仓 | 适用场景 |
|------|------|---------|----------|
| **仓库存储** | 供应商发货至我方仓库，入库上架后待下游要货再出库配送 | 是 | 提前备货 |
| **即入即出** | 供应商发货至我方仓库，即入即出后立即配送至终端 | 是 | 无备货、不在院端配送清单 |
| **院端配送** | 供应商直接发货至终端，不经我方仓库 | 否 | 在院端配送清单内 |
| **设备直放** | 供应商收到直放通知单后直接发货至终端 | 否 | 设备项目 |
| **直调** | 供应商直接发货至指定地点 | 否 | 紧急情况（灾情、疫情、临床紧急救治） |

### 1.4 项目类型

| 类型 | 定义 |
|------|------|
| **GPO项目** | 在终端集中采购配送招标中中标，已签订GPO合同或取得中标通知书 |
| **类GPO项目** | 不满足GPO条件但立作GPO项目，按产品线项目规范执行 |

### 1.5 业务接管

上游供应商已将医疗器械暂存于医疗机构或ICL等终端用户，因业务调整，我方向上游供应商采购暂存存货。

## 二、应收业务

### 2.1 应收单来源

```
销售订单 → 产品出库 → 金蝶自动生成应收单（已审核）
```

### 2.2 应收单类型与凭证规则

| 单据类型 | 是否生成凭证 | 会计分录 |
|----------|-------------|----------|
| 经销销售应收 | 是 | 借：发出商品 / 贷：库存商品-货值 |
| 经销销售应收（进口） | 是 | 借：发出商品 / 贷：库存进口商品-货值、库存进口商品-关税 |
| 经销销售应收（有返利） | 是 | 借：发出商品 / 贷：库存商品-货值 + 借：发出商品 / 贷：其他应付款-返利 |
| 寄售销售应收 | 是 | 借：委托代销商品-发出商品-客户 / 贷：委托代销商品-库存商品 |
| 设备类应收 | 是 | 借：发出商品 / 贷：库存商品-货值 |
| 应收退回（寄售/经销） | 否 | - |
| 修订应收（寄售/经销） | 否 | - |
| 以销订购 | 否 | - |
| 暂存核销（单号含SA） | 否 | - |

### 2.3 暂存出库单

**定义**：货品已发出但暂不满足收入确认条件，货品所有权仍归销售方。

**出库类型**：暂存出库、寄售调出、样品出库、销售换出、赠品出库

| 出库类型 | 货品类型 | 会计分录 |
|----------|----------|----------|
| 暂存出库 | 寄售 | 借：受托代销商品-发出商品-客户 / 贷：受托代销商品-库存商品 |
| 暂存出库 | 经销（无返利） | 借：发出商品 / 贷：库存商品-货值 |
| 暂存出库 | 经销（有返利） | 借：发出商品 / 贷：库存商品-货值 + 借：发出商品（红字） / 贷：其他应付款-返利 |
| 寄售调出（标准成本） | 寄售 | 借：受托代销商品款-供应商 / 贷：受托代销商品-库存商品 |
| 寄售调出（实际成本） | 经销 | 借：受托代销商品款-供应商 / 贷：受托代销商品-库存商品 |
| 赠品出库 | - | 视同销售，按13%税率拆分不含税金额和税额入账 |

### 2.4 暂存入库单

**定义**：采购商品已入库，财务还未确认应付金额，已取得采购商品控制权。

**入库类型**：暂存调回入库、寄售入库、销售换入、进口换货入库

**成本计算规则**：
- 寄售入库：按标准成本
- 暂存出库/核销（寄售货）：按标准成本
- 暂存出库/核销（经销货）：按不含税实际成本
- 退货：同上述规则

## 三、收款认领

### 3.1 业务流程

```
客户打款 → 金蝶生成收款单 → 推送至核心平台 → 商务认领收款
```

### 3.2 认款类型

| 认款类型 | 适用场景 | 是否需审批 |
|----------|----------|-----------|
| 认款到订单 | 预收账期订单，先打款后发货 | 否（直接完成） |
| 认款到发票 | 收货账期或其他非预收账期 | 是（金蝶审核） |
| 认款到初始应收 | 旧系统未冲销的应收 | 是 |
| 认款到负数应收 | 客户有负数应收需抵用 | 否 |
| 暂收款 | 客户打款后暂不发货 | 否 |

### 3.3 认款操作规则

| 规则 | 说明 |
|------|------|
| 同一认款单类型限制 | 不可同时认款到发票、订单、初始应收，只能选一种 |
| 部分认款 | 应收单部分认款后，剩余部分可通过其他方式认款 |
| 盘点期间限制 | 盘点期间不可认款 |
| 三方回款 | 需在客户管理中设置"三方回款"为是 |
| 跨月撤销 | 支持跨月撤销认款 |

### 3.4 暂收款处理

```
创建暂收款 → 添加明细（项目、收款类型、金额） → 提交 → 转货款 → 正常认款
```

**注意事项**：
- 暂收款可指定多个客户及各自可认款金额
- 暂收款转货款前，第一次认款需完成
- 暂收款撤销时会删除所有未提交的认款单

## 四、应付业务

### 4.1 应付单来源

| 来源 | 说明 |
|------|------|
| 采购入库 | 经销采购入库后生成 |
| 寄售转购 | 寄售货物销售结算后生成 |
| 购货修订 | 修订采购价格产生差异 |
| 销售调回 | 调回后调出或重新入库产生负数应付 |

### 4.2 付款计划

| 账期类型 | 付款时机 | 说明 |
|----------|----------|------|
| 预付账期 | 采购前 | 需先付款再采购 |
| 入库账期 | 入库后N天 | 货物入库后按约定天数付款 |
| 销售账期 | 销售后N天 | 货物销售后按约定天数付款 |
| 回款账期 | 收款后N天 | 收到下游货款后按约定天数付款 |
| 验收账期 | 验收后N天 | 设备验收后按约定天数付款 |

### 4.3 应付冲销规则

| 冲销类型 | 条件 | 操作路径 |
|----------|------|----------|
| 正负应付冲销 | 同项目、同业务单元、同供应商、同公司 | 应付清单 → 进行冲销 |
| 应收抵应付 | 同公司、同项目、同统一信用代码的客户和供应商 | 应付清单 → 进行冲销 |
| 负数应付与收款单冲销 | 同公司、同供应商、同项目 | 应付清单 → 进行冲销 |
| 正数应付与付款单冲销 | 同公司、同供应商、同项目 | 应付清单 → 进行冲销 |

**冲销优先级**：预付账期 > 入库账期 > 销售账期 > 回款账期

**撤销限制**：
- 仅支持当月冲销撤回
- 仅支持一对一整单撤回
- 批量付款和预付冲销不能撤回

## 五、寄售垫资

### 5.1 垫资定义

寄售业务中，我方先向上游付款，等待下游回款的资金占用期间产生的利息收入。

### 5.2 核心指标

| 指标 | 计算公式 |
|------|----------|
| 垫资天数 | 医院回款天数 - 供应商付款天数 |
| 垫资日利率 | 实际供应链金融折扣 / 垫资天数 |
| 资金占用金额 | 逾期状态为"未回款"的应付金额合计 |
| 垫资利息收入 | 应收金额 / (1+销售税率) × 实际供应链金融折扣 |
| 基础毛利 | 应收金额 / (1+销售税率) × 基础折扣 |
| 合计毛利 | 基础毛利 + 垫资利息收入 |

### 5.3 垫资状态

| 状态 | 条件 |
|------|------|
| 未垫资 | 付款单号为空 |
| 未垫资(提前回款) | 付款单号为空，收款日期不为空且小于垫资应收到期时间 |
| 垫资 | 付款单号不为空 |

### 5.4 逾期状态

| 状态 | 条件 |
|------|------|
| 应收未逾期未垫资 | 付款金额=0 |
| 应收已逾期未回款 | 收款日期为空，逾期天数>0 |
| 应收已逾期已回款 | 收款日期不为空，逾期天数>0 |
| 应收未逾期未回款 | 收款日期为空，逾期天数<=0 |
| 应收未逾期已回款 | 收款日期不为空，逾期天数<=0 |

### 5.5 逾期区间

| 区间 | 条件 |
|------|------|
| 逾期90天以内(含) | 0 < 逾期天数 <= 90 |
| 逾期90-180天(含) | 90 < 逾期天数 <= 180 |
| 逾期180-365天(含) | 180 < 逾期天数 <= 365 |
| 逾期365天以上 | 逾期天数 > 365 |

## 六、开票业务

### 6.1 开票流程

```
应收单生成 → 运营提交开票 → 制作开票 → 金蝶审核 → 开票完成
```

### 6.2 红冲场景

| 场景 | 操作方式 |
|------|----------|
| 客户接受原蓝票直接开红票 | 运营制作开票 → 提交负数应收 → 编辑填蓝票信息 → 开红票 |
| 新系统单据不接受原票重开 | 提交开票 → 找原正数应收 → 红冲发票 → 还原待开票 → 重新开票 |
| 老系统蓝票换票不换货 | 订单初始化 → 订单修订（负数）→ 开红票 → 订单修订（正数）→ 开蓝票 |

### 6.3 开票规则

| 规则 | 说明 |
|------|------|
| 数电票红冲 | 需先在核心平台做红字确认单 |
| 发票备注长度 | 限制200字符 |
| 折扣行 | 售价为0时可新增折扣行 |
| 税收编码 | 需设置9位数字分类编码 |
| 免审条件 | 非红票、未修改开票名称/规格/税率 |

## 七、配送流程

### 7.1 即入即出流程

```
下游要货 → 向供应商发送要货明细 → 供应商发货至我方仓库 
→ 入库申请-即入即出 → 货品扫码确认 → 审批 
→ 自动生成出入库单据 → 货品发出 → 下游签收 → 上传签收单
```

**寄售与经销差异**：
- 寄售：入库申请-即入即出，生成应收单据
- 经销：采购订单-即入即出，生成应付单据+应收单据

### 7.2 院端配送流程

```
下游要货 → 向供应商发送要货明细 → 供应商预留货物并发送明细
→ 申请入库申请/采购订单-院端配送 → 审批 
→ 下达发货指令 → 供应商直接发货至终端 → 下游签收 → 上传签收单
```

### 7.3 设备直放流程

```
下游提出设备需求 → 设备项目立项 → 推送采购订单 → 推送销售预订单
→ 预订单转直放订单 → 审批生成直放通知单 → 供应商发货至终端
→ 直放细化（维护批号效期）→ 生成出入库单据 → 下游签收 → 上传验收报告
```

### 7.4 业务接管流程

```
接收接管通知 → 盘库（录制视频）→ 暂存初始化单头申请 → 审批
→ 确认交接明细 → 创建暂存初始化数据 → 审批 → 生成暂存数据
```
