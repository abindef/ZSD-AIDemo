# 财务问题解决方案

本文档整理财务模块常见问题及解决方案，按业务场景分类。

---

## 一、开票问题

### 1.1 红冲开票

| 问题场景 | 解决方案 |
|----------|----------|
| 新系统单据，客户接受原蓝票直接开红票 | 财务管理 → 运营制作开票 → 提交负数应收 → 点击【编辑】填蓝票信息 → 开红票 |
| 新系统单据，不接受原票想重新开票 | 财务管理 → 提交开票 → 找原正数应收 → 点击【红冲发票】→ 还原待开票 → 重新开票 |
| 老系统蓝票，换票不换货（不销售调回） | 做订单初始化 → 订单修订（负数金额）→ 开红票 → 再次订单修订（正数金额）→ 开蓝票 |
| 老系统蓝票，换票不换货（不走订单修订） | 金蝶直接开红票和蓝票 → 邮件通知运维后台修改发票号码和开票日期 |
| 老系统经销货，部分/全部调回重开票 | 订单初始化 → 销售调回入库申请 → 重新销售出库 → 用负数应收开红票 → 用新应收开蓝票 |
| 老系统寄售货（非B类），部分/全部调回 | 订单初始化 → 销售调回入库申请（设置"重新入库"）→ 重新销售出库 → 开红票蓝票 → 正负应付冲销 → 正负应收认款冲销 |
| 老系统寄售货（B类），部分/全部调回 | 订单初始化 → 销售调回入库申请（设置"调回后调出"）→ 重新做B类订单 → 开红票蓝票 → 正负应付冲销 → 正负应收认款冲销 |

### 1.2 开票报错

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 设备编码未找到 | 纸质票/电子票需要设备编码 | 点击【编辑】修改开票类型为其他类型 |
| 选不到开票主体信息 | 客户开票信息未维护 | 基础数据 → 客户管理 → 基本信息维护 |
| 税收编码不一致/为空 | 税收编码未设置或格式错误 | 设置9位数字分类编码 → 基础数据维护税收分类编码 → 同步税收分类编码 |
| 合并失败，负数商品冲抵不了 | 销售调回不是折扣行 | 金蝶退回 → 提交开票点击【合并明细】→ 重新提交 |
| 开红票报税收编码不一致 | 金蝶红字没有税收编码 | 联系金蝶运维协助处理 |

### 1.3 开票操作

| 操作需求 | 解决方案 |
|----------|----------|
| 开折扣行 | 销售单有原价售价为0的货品时，运营提交开票会出现【新增折扣行】按钮 |
| SPD不同单价合并开票 | 制作页面获取所有明细 → 修改单价一致 → 点击"按产品名称，单价合并明细" |
| 一个订单开多张发票 | 开票界面点击明细 → 设置为另一个开票单 |
| 部分数据先不开票 | 设置为另一个开票单 → 再设置为另一个开票分类 → 新分类不予开票 |
| 发票备注过长 | 系统限制200字符，可找金蝶设置票面显示银行信息 |

### 1.4 开票撤回

| 状态 | 操作 |
|------|------|
| 草稿 | 更多操作 → 删除 |
| 审批中 | 更多操作 → 撤回 |
| 已审批未开票 | 金蝶退回（列表点反审核 → 进单据点打回）→ 核心重新操作 |
| 已完成 | 不能撤回，需走红冲流程 |

## 二、付款问题

### 2.1 批量付款报错

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| OA审批报没有银行信息 | 供应商未维护收款信息 | 检查供应商收款信息 → OA驳回 → 设置供应商收款账号 → 重新提交 |
| OA审批报项目没有 | 初始化应付项目名称问题 | 用ID查应付项目名称 → 查项目运行清单 → 联系运维修改项目名称 |
| 协议未回签 | 供应商协议未回签 | 项目管理 → 子任务清单 → 所有该供应商协议回签，或申请延期 |
| 项目没有付款金额 | 采购额度配置不足 | 运营管理 → 检查项目采购额度配置 → 调整金额 |

### 2.2 付款操作

| 操作需求 | 解决方案 |
|----------|----------|
| 找不到应付单 | 1. 检查是否已创建批量付款 2. 检查账期是否已到期 3. 检查付款计划预计付款日期是否存在 |
| 只支付部分款 | 不涉及垫资的应付单，付款计划右下角【拆分】按钮进行拆分 |
| 汇票付款/提前贴现 | 金蝶出纳模块操作 |
| 应收冲销应付后不能付款 | 正负数冲销的应收对应正数应付付款计划无预计付款日期，回款账期有预计付款日期才能付款 |
| 正负应收冲销后付款 | 检查应收单冲销记录 → 只能付两个销售单正负数应付抵冲后的金额 |

### 2.3 应付冲销

| 冲销类型 | 条件 | 操作 |
|----------|------|------|
| 正负应付冲销 | 同项目、同业务单元、同供应商、同公司 | 应付清单 → 进行冲销 |
| 非同业务单元冲销 | 业务单元同意+领导同意 | 走邮件联系运维后台处理 |
| 同供应商不同项目不同业务单元 | 特殊审批 | 走邮件联系运维后台处理 |
| 正数应付与正数应收冲销 | 同公司、同项目、同统一信用代码 | 应付清单 → 进行冲销（服务费应收抵货款） |
| 负数应付与收款单冲销 | 同公司、同供应商、同项目 | 应付清单 → 进行冲销 |

**冲销优先级**：预付账期 > 入库账期 > 销售账期 > 回款账期

**撤销限制**：仅支持当月一对一整单撤回，批量付款和预付冲销不能撤回

### 2.4 负数应付来源

检查以下情况：
1. 入库申请是否为"调回后调出"或"重新入库"
2. 是否做过采购-购货修订
3. 是否做过退货出库（采购单退货给供应商）
4. 是否做过销售-订单修订（降成本）

## 三、认款问题

### 3.1 认款操作

| 认款类型 | 操作说明 |
|----------|----------|
| 负数应收认款 | 与正常收款单相同方式 |
| 未知款项认款 | 先指定收款类型（预收款或销售回款）再认款 |
| 认款到发票 | 单号填发票号 → 需金蝶审核 |
| 认款到初始应收 | 单号填应收单号（旧系统未冲销的应收） |
| 认款到销售订单 | 单号填销售单号（新系统单据） |

### 3.2 认款撤销

| 场景 | 是否可撤销 |
|------|-----------|
| 金蝶已结算 | 可以，核心点击撤回认款 |
| 认款初始应收 | 不支持撤回 |
| 销售单未审核或驳回 | 可以撤回 |
| 跨月认款 | 支持跨月撤回 |
| 对应应付已付款 | 可以撤销，但不建议（付款完成记录无法撤销） |

### 3.3 认款失败排查

| 错误场景 | 解决方案 |
|----------|----------|
| 销售单和收款单客户名称不一致 | 删除认款单 → 找金蝶处理收款单和客户 → 收款单生成凭证 → 重新认款 |
| 收款客户和实际客户不一致（三方回款） | 删除认款单 → 基础数据客户管理设置"三方回款"为是 → 重新认款 |
| 认款直接报错（客户id不存在） | 删除认款单 → 找金蝶处理收款单和客户 → 收款单生成凭证 → 重新认款 |
| 一个认款单一张发票认多次 | 一张发票只能认一次，或建多个认款单认同一发票 |
| 认款红票失败 | 新系统不支持认款红票，负数应收先认到订单号 → 客户实际打款再认到蓝票 |
| 应收全部冲销后认款 | 无法认款，应收全部冲销后不能再认款 |
| 提示备注必填 | 同客户多张票跳号开票，需开启跳号并填写备注 |
| 结算金额不等于认款金额 | 检查认款明细是否有结算的应收，没有需删除重新添加 |

### 3.4 认款未冲销

| 原因 | 解决方案 |
|------|----------|
| 金蝶回调定时任务 | 存在半小时时间差，等待调度执行 |
| 一直未冲销 | 联系运维排查 |

## 四、进项票问题

| 问题场景 | 解决方案 |
|----------|----------|
| 包含旧系统入库单数据 | 页面先勾选新系统数据 → 联系运维后台处理（采购、财务、金蝶） |
| 旧系统数据后来做了购货修订 | 页面先勾发票绑负数应付 → 联系运维后台处理 |
| 非业务进项票 | 财务评估后点击右上角【忽略发票】 |
| 勾稽报发票号码未找到对应收票单 | 金蝶【应付】→【发票处理】→【收票单处理】先下载发票 |
| 红票蓝票相加总额绑同一订单 | 页面暂不支持，联系运维后台处理 |
| 勾稽后蓝票重开 | 页面点击【取消勾稽】→ 重新勾稽新蓝票 |

## 五、垫资与发票入账

### 5.1 发票入账

| 问题 | 解决方案 |
|------|----------|
| 销售账期付款提示未入账 | 财务管理 → 发票入账单 → 将对应发票入账 |
| 发票入账单无客户/业务单元 | 项目管理 → 寄售垫资运行清单查询是否有垫资规则且要求发票入账 |

**发票入账前提**：
1. 项目管理 → 寄售垫资运行清单查看是否有寄售垫资单
2. 确认垫资规则是否允许发票入账

### 5.2 垫资相关

发票入账是确认发票已成功送达医院并完成入账，防止因发票丢失导致回款不正常，影响资金流动。

## 六、其他问题

### 6.1 收款单问题

| 问题 | 解决方案 |
|------|----------|
| 客户付款了核心查不到收款单 | 1. 确认金蝶是否有收款单 2. 财务在金蝶点确认收款 3. 用收款日期在待认款查找 4. 联系金蝶运维 |
| 客户打错款需退款 | 财务管理 → 退款处理 → 选择"客商退我方"或"我方退客商" |

### 6.2 红字确认单

| 问题 | 解决方案 |
|------|----------|
| 搜不到单据 | 旧系统蓝票没有单据，需金蝶做红字确认单 |
| 报原发票错误 | 需填蓝字代码 |

### 6.3 集团寄售

| 问题 | 解决方案 |
|------|----------|
| 母公司没有生成应收数据 | 1. 排查是否成功生成寄售转购货单据 2. 联系运维协助处理 |

### 6.4 财务盘点

| 问题 | 解决方案 |
|------|----------|
| 盘点设定完成失败 | 需要其他盘点单都生成成功 |
| 对账管理创建报没有月度盘点 | 需要先完成财务总盘点 |
| 财务总盘点报返利计提单未完成 | 检查对应公司返利计提单状态，找创建人完成 |

### 6.5 跨月平账

**场景**：6月500元已开蓝票，7月有退货或发现错误，正确应为400元

**解决方案**：保持7月红票蓝票对冲后为-100红票，7月正负应收对冲后为-100

### 6.6 权限开通

OA流程 → 新建流程 → 人事管理：系统身份开通/变更/关闭流程
